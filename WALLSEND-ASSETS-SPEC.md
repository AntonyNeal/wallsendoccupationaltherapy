## üìã Delivery Specifications

### File Naming Convention

```
[asset-type]-[description]-[variant].[ext]

Examples:
- logo-wallsend-ot-horizontal.svg
- og-image-wallsend-ot.jpg
- service-ndis-assessment.jpg
- team-sarah-chen.jpg
```

### Folder Structure

```
public/
  assets/
    logos/
      logo-wallsend-ot.svg
      logo-wallsend-ot-white.svg
      logo-wallsend-ot-icon.svg
    favicons/
      favicon.svg
      favicon-32x32.png
      favicon-16x16.png
      apple-touch-icon.png
    social/
      og-image-wallsend-ot.jpg
      twitter-card-wallsend-ot.jpg
    hero/
      hero-wallsend-ot.jpg
      hero-clinic.jpg (variant)
    services/
      service-ndis-assessment.jpg
      service-home-modifications.jpg
      [etc...]
    team/
      team-sarah-chen.jpg
      team-michael-thompson.jpg
      team-emma-rodriguez.jpg
    about/
      clinic-reception.jpg
      therapy-room.jpg
```

### Optimization Requirements

- **Images**: Run through TinyPNG or ImageOptim
- **SVGs**: Clean up with SVGO, remove unnecessary metadata
- **WebP**: Consider providing WebP versions for modern browsers
- **Lazy Loading**: Ensure images are optimized for lazy loading

### Accessibility Requirements

- All images MUST have descriptive alt text
- Decorative images should have empty alt=""
- Complex images (infographics) need detailed descriptions
- Ensure sufficient color contrast on overlaid text
